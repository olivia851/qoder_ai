# 开发指南文档

此目录包含开发过程中的指导文档和规范。

## 文档类型

- 编码规范和风格指南
- 开发环境搭建说明
- 代码审查标准
- 分支管理和合并策略
- 版本发布流程
- 开发工具使用指南

## 重要原则

所有文档必须遵循项目宪法中的编码规范要求：
- 命名必须清晰表达业务语义
- 遵循单一职责原则
- 保持代码可测试性
- 注重用户体验

- `coding-standards.md` - 编码规范
- `dev-environment-setup.md` - 开发环境配置
- `code-review-guidelines.md` - 代码审查指南
- `branch-management.md` - 分支管理策略
